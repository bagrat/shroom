# uploader tests — provenance of the SigV4 golden vectors

`sigv4.test.mjs` is a **known-answer test**: the expected `Authorization` headers
are not hand-computed or remembered — they were generated by **AWS's own reference
signer**, `botocore.auth.S3SigV4Auth`, under a frozen clock. So the test asserts
our Node signer reproduces AWS byte-for-byte; if R2 accepts botocore, it accepts us.

To regenerate or extend the vectors (e.g. new key shapes), drive real botocore in a
throwaway virtualenv:

```bash
python3 -m venv /tmp/oraclevenv
/tmp/oraclevenv/bin/pip install botocore
```

```python
# oracle.py — sign with AWS's reference implementation under a frozen clock
import json, datetime, hashlib
import botocore.auth as bauth
from botocore.auth import S3SigV4Auth
from botocore.awsrequest import AWSRequest
from botocore.credentials import Credentials

FROZEN = datetime.datetime(2026, 1, 2, 3, 4, 5, tzinfo=datetime.timezone.utc)
class _Frozen(datetime.datetime):
    @classmethod
    def utcnow(cls): return FROZEN.replace(tzinfo=None)
bauth.datetime.datetime = _Frozen

req = AWSRequest(method="PUT",
    url="https://acct123.r2.cloudflarestorage.com/mybucket/AbC123/seg_00000.m4s",
    data=b"hello-shroom-segment", headers={})
S3SigV4Auth(Credentials("AKIAIOSFODNN7EXAMPLE",
    "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"), "s3", "auto").add_auth(req)
print(req.headers["X-Amz-Date"], req.headers["Authorization"])
```

Paste the emitted `Authorization` (and `X-Amz-Date`) into `sigv4.test.mjs`. Always
regenerate from botocore first, then make the Node code match — never edit an
expected value to match our output.

`uploader.test.mjs` needs no external oracle: it runs the `Uploader` against the
in-process `mock-s3.mjs` (PUT/HEAD/GET + failure injection), all on loopback.
